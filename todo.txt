MULTIPLAYER 2D CANVAS GAME - STEP BY STEP PLAN
==============================================

PHASE 1: PROJECT SETUP
----------------------
[ ] 1. Update package.json with dependencies:
       - express (HTTP server)
       - socket.io (Real-time WebSocket communication)
       - socket.io-client (Client-side Socket.IO)
       - typescript, esbuild (Build tools)
       - @types/express, @types/node (TypeScript types)
       - concurrently, nodemon (Dev tools)

[ ] 2. Configure TypeScript (tsconfig.json)
       - Set up for both server (Node.js) and client (browser) code
       - Output to dist/ directory
       - Source in src/ directory


PHASE 2: SHARED CODE
--------------------
[ ] 3. Create shared types (src/shared/types.ts)
       - Player interface (id, position, velocity, color, name, size)
       - InputState interface (up, down, left, right)
       - GameState interface
       - Socket.IO event types (ServerToClientEvents, ClientToServerEvents)
       - Game constants (TICK_RATE, PLAYER_SPEED, WORLD_WIDTH, WORLD_HEIGHT)


PHASE 3: SERVER-SIDE
--------------------
[ ] 4. Create Express + Socket.IO server (src/server/index.ts)
       - Set up HTTP server with Express
       - Initialize Socket.IO for WebSocket connections
       - Serve static files (HTML, client JS)
       - Start game server

[ ] 5. Create GameServer class (src/server/GameServer.ts)
       - Manage connected players (Map)
       - Handle player input
       - Run game loop (60 ticks/second)
       - Update player positions based on input
       - Keep players within world bounds
       - Broadcast game state to all clients
       - Handle player join/leave events


PHASE 4: CLIENT-SIDE
--------------------
[ ] 6. Create HTML page (public/index.html)
       - Canvas element
       - CSS styling
       - Load Socket.IO client library
       - Load game bundle

[ ] 7. Create client entry point (src/client/index.ts)
       - Wait for DOM ready
       - Initialize Game class
       - Handle window resize

[ ] 8. Create Game class (src/client/Game.ts)
       - Initialize canvas and 2D context
       - Create NetworkManager, InputManager, Renderer
       - Set up network event handlers
       - Run render loop (requestAnimationFrame)
       - Send input to server each frame
       - Render all players

[ ] 9. Create InputManager (src/client/InputManager.ts)
       - Listen for keydown/keyup events
       - Track pressed keys (Set)
       - Support WASD and Arrow keys
       - Return current InputState

[ ] 10. Create NetworkManager (src/client/NetworkManager.ts)
        - Connect to server via Socket.IO
        - Send player input to server
        - Receive game state updates
        - Handle welcome, playerJoined, playerLeft events
        - Callback system for events

[ ] 11. Create Renderer (src/client/Renderer.ts)
        - Draw world boundary
        - Draw grid background
        - Draw players (circles with shadows, highlights)
        - Draw player names
        - Draw UI (player count, controls, connection status)


PHASE 5: BUILD & RUN
--------------------
[ ] 12. Set up build scripts in package.json
        - build:server - Compile TypeScript for server
        - build:client - Bundle client code with esbuild
        - build - Run both builds
        - start - Run compiled server
        - dev - Watch mode for development

[ ] 13. Install dependencies
        npm install

[ ] 14. Build the project
        npm run build

[ ] 15. Run and test
        npm start
        Open http://localhost:3000 in multiple browser tabs


ARCHITECTURE OVERVIEW
=====================

SERVER
------
Express (HTTP) + Socket.IO (WebSocket)
    │
    ▼
GameServer
    - Game loop running at 60 ticks/second
    - Manages all connected players
    - Processes input from clients
    - Updates game state (positions, collisions)
    - Broadcasts state to all clients

CLIENT
------
HTML Canvas
    │
    ▼
Game (main controller)
    ├── NetworkManager (Socket.IO connection)
    ├── InputManager (keyboard input)
    └── Renderer (canvas drawing)
